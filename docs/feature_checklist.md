- [x] **세부 페이지 카드형 UI 개편 (면적당 공사비 검색 등)**
- [x] **초록색 띠 제거 및 다크 네이비 푸터 통일**
- [x] **정적 콘텐츠 페이지들 (이용약관, 개인정보처리방침, 오시는길, 회사소개 등)**

## 2. 백엔드(API) 및 DB 연동 필요 기능
GitHub Pages는 단순 정적 문서만 띄워주기 때문에, 아래의 핵심 비즈니스 로직들은 현재 "빈 결과 상태(Empty State)" 또는 "링크 이동" 수준으로 처리되어 있습니다. **별도의 API 서버(Node.js, PHP 등)와 클라우드 스토리지 연동이 필수**적입니다.

### 📊 공사비 데이터 검색 
- **면적당 공사비 검색**: UI 카드형태로 반영 완료 / 데이터는 DB 필요
- **건물종류별/지역별 검색**: 링크 있음 / 데이터는 DB 필요
- **세부 공종별/단가 검색**: 링크 있음 / 데이터는 DB 필요

### 📋 내역서 작성 엔진
- **건축 내역서 / 수량 산출**: UI만 반영 (단계형 UI) / 계산 로직 및 DB 저장 기능 필요

### 🎓 공사비 교육
- **동영상 강의 / 교재 구매**: 링크 및 카드 UI 있음 / 실제 영상 스트리밍 및 결제 모듈 연동(API) 필요

### 💬 커뮤니티 및 게시판
- **공지사항 / 자료실 / Q&A**: 링크 있음 / PHP 또는 API 기반의 게시글 조회 및 첨부파일 다운로드 로직 필요

### 🔐 인증 및 관리자
- **로그인 / 회원가입 / 마이페이지**: UI 화면 있음 / 세션 관리, 암호화, 회원 DB 연동 필수
- **관리자 페이지**: 현재 소스에는 제외됨 (보안 분리 필요)

## 3. 보안 아키텍처 전환 가이드 (Cloud-Ready)
이 저장소를 향후 온전한 서비스로 구동하기 위해서는 다음 단계를 따라야 합니다.
1. **스토리지 연동**: `public/static/js/storage.config.js` 에 클라우드 버킷(S3 등) 주소를 입력합니다. 백업본의 `file` 또는 `upload` 폴더 내용물들을 이 버킷에 업로드하세요.
2. **백엔드 구축**: Node.js/Express 또는 기존 PHP 서버를 구동하고 API 주소를 부여합니다.
3. **환경변수 세팅**: `.env.example` 파일을 복사하여 실제 DB 접속 정보와 API 경로를 담은 `.env` 파일을 서버에 생성합니다. (GitHub에는 절대 업로드 불가)
